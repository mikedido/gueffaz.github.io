Mahdi GUEFFAZ

Web developper & DevOps engineer

Bonjour! Je m'appelle Mahdi (prononcé "Medi"), un développeur de logiciels avec plus de 6 ans d'expérience dans un large éventail de disciplines. Actuellement, je suis développeur Symfony back-end avec Angular/React comme front-end chez GRDF. Les developpements des scripts d'automatisation des migrations, mises en productions, analyses des données sont en Bash/ Python. Mon objectif est de guider mon expérience dans la méthodologie DevOps.

Formation

J'ai obtenu mon diplôme d'ingénieur en informatique à l'Université d'Alger (USTHB) en 2008. Après cela, j'ai obtenu mon master en traitement d'image à l'Université de Bourgogne. Cela m'a permis d'avoir quelques idées sur la composition des images. Enfin, j'ai également obtenu mon doctorat à l'Université de Bourgogne dans la recherche en ontologie sémantique. Je développe vraiment mon expérience et prends un peu de recul sur mes choix de conception.

Nouvelle technologie

PHP, mysql, javascripts, linux, ....

Je suis actuellement employé en tant que développeur back-end/front-end chez GRDF. Basé sur le framework Symfony pour le développement d'une application de gestion de la demande des utilisateurs pour leurs besoins en gaz. Auparavant, j'ai travaillé 3 ans chez Orange Cyber Defense en utilisant le framework Zend pour le développement d'applications évitant les défaillances de sécurité dans les logiciels de nos clients.

Pour tous ces projets, j'utilise Git pour suivre les changements de code source lors du développement logiciel, Jenkins comme logiciel d'intégration continue au développement logiciel, Apache comme serveur web. Sans oublier Docker pour notre environnement de développement et pour le déploiement. MySQL est le système de gestion de base de données relationnelles classiques pour nos données dans les bases de données. Linux est mon meilleur système d'exploitation. Cela vous permet de mieux comprendre votre machine ... Et quelques développements en Bash et en Python pour automatiser nos services et aussi pour le plaisir.

Java, OWL, RDF ...

Quand j'étais à Telecom ParisTech, j'ai travaillé sur le projet ILOT. Il s'agit d'une collaboration de projet qui combine Gutenberg technology, LeLivreScolaire, Alcatel Lucent pour offrir un entrepôt de ressources pédagogiques permettant le développement collaboratif de ces ressources, leur annotation et leur exploitation dans différents modes de consultation.

J'ai également travaillé sur le projet ScaleSem pendant mon doctorat. ScaleSem est un sous-projet du projet CheckSem qui est en architecture pour apporter des réponses à une multitude de problèmes liés à la gestion des connaissances. ScaleSem est situé dans la couche Back-Office de l'architecture CheckSem, qui constitue la base du modèle CheckSem. Cette partie, utilisateurs non visibles du système, propose des fonctions permettant d'extraire les connaissances contenues dans toutes sortes de structures ou de traduire les connaissances d'un formalisme sémantique donné à un autre.

DevOps, Jenkins .... CI/CD

DevOps, cette culture que j'ai appris lors du cycle de developpement des applications m'a permis l'automatisation de nos mises en production en respectant la chaîne de l'intégration continue avec des outils comme Jenkins/Gitlab CI. Ce flux comprenait, des tests unitaires (PHPUNIT, unitest), fonctionnels....du code coverage....de la review de code, ...etc
